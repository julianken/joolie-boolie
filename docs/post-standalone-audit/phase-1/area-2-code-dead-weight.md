# Investigation: Area 2 â€” Code-Level Dead Weight

## Summary

The source code of both apps and all 8 shared packages is clean of live imports from deleted packages (`@joolie-boolie/auth`, `@joolie-boolie/database`, `@joolie-boolie/platform-hub`). No auth hooks, auth stores, sign-in forms, or OAuth flows remain in `apps/bingo/src` or `apps/trivia/src`. The dead weight that does remain falls into four categories: (1) stale env vars baked into committed `.env.local` files that load auth-era secrets into running dev processes, (2) auth-world types still exported from `packages/types` that are never imported by any consumer, (3) a ghost Supabase mock section in `packages/testing`'s README that documents an API that was never implemented in the actual source, and (4) dead operational artifacts â€” a committed `.env` file with live Supabase credentials, `start-e2e-servers.sh` that still tries to start `platform-hub`, and a nightly CI workflow still injecting five defunct env stubs at build time.

## Key Findings

### Finding 1: Committed `.env` at repo root contains live Supabase credentials and auth secrets
- **File:** `/Users/j/repos/beak-gaming-platform/.env:1-39`
- **Evidence:** File contains `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`, `SUPABASE_SERVICE_ROLE_KEY`, `SESSION_TOKEN_SECRET`, `NEXT_PUBLIC_OAUTH_CLIENT_ID`, `NEXT_PUBLIC_OAUTH_REDIRECT_URI`, `NEXT_PUBLIC_OAUTH_CONSENT_URL` â€” all with real credential values (not placeholders). This file is committed to git (not in `.gitignore`).
- **Severity:** critical
- **Confidence:** high
- **Impact:** Live Supabase service role key and session secret are in git history and in every developer's working directory. Any process reading root `.env` (e.g., Playwright global setup, scripts sourcing the file) picks up these auth-era values, which is also confusing for new contributors.
- **Recommended action:** Rotate all secrets in this file immediately. Replace the entire file with only the keys still needed (none for auth; at most `TURBO_TOKEN`/`TURBO_TEAM` if used). The existing `/.env.example` is clean and should remain.

### Finding 2: Both `.env.local` files in apps contain a full auth-era credential set
- **File:** `/Users/j/repos/beak-gaming-platform/apps/bingo/.env.local:1-13` and `apps/trivia/.env.local:1-14`
- **Evidence:** Both files contain `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`, `SUPABASE_SERVICE_ROLE_KEY`, `SESSION_TOKEN_SECRET`, `NEXT_PUBLIC_PLATFORM_HUB_URL`, `NEXT_PUBLIC_OAUTH_CLIENT_ID`, `NEXT_PUBLIC_OAUTH_REDIRECT_URI`, `NEXT_PUBLIC_OAUTH_CONSENT_URL`, `E2E_JWT_SECRET`, and `SUPABASE_JWT_SECRET` â€” with real credential values. `apps/trivia/.env.local` also has `E2E_TESTING=true` absent (bingo has it set to `true` permanently on line 11).
- **Severity:** critical
- **Confidence:** high
- **Impact:** The real `SUPABASE_SERVICE_ROLE_KEY` (admin-level Supabase key) is readable by any process. `E2E_TESTING=true` permanently set in bingo `.env.local` means every local dev run runs in E2E bypass mode, disabling Sentry (`sentry.server.config.ts:4` checks `E2E_TESTING`). - **Recommended action:** Rotate the service role key and JWT secret. Replace both `.env.local` files with only the vars the apps actually read: `NEXT_PUBLIC_APP_URL`, `SENTRY_DSN` (optional), `OTEL_*` (optional), and `THE_TRIVIA_API_KEY` (trivia only). Remove `E2E_TESTING=true` from bingo's `.env.local`. Add `apps/bingo/.env.local` and `apps/trivia/.env.local` to `.gitignore` if not already there.

### Finding 5: `packages/types/src/user.ts` exports a full auth-world type surface that no consumer imports
- **File:** `/Users/j/repos/beak-gaming-platform/packages/types/src/user.ts:1-93`
- **Evidence:** Exports `User`, `UserProfile`, `LoginRequest`, `RegisterRequest`, `AuthResponse`, `UpdateProfileRequest`, `Session`. All are re-exported from `packages/types/src/index.ts:40-48`. - **Severity:** high
- **Confidence:** high
- **Impact:** Dead exported surface that signals the platform still has auth concerns. - **Recommended action:** Delete `packages/types/src/user.ts` and `packages/types/src/api.ts`. Remove their re-exports from `packages/types/src/index.ts`. Note: `apps/bingo/src/types/index.ts` has its own local `ApiResponse<T>` and `PaginatedResponse<T>` definitions (lines 296-307) that are also unused (no importer in bingo src) â€” those should be deleted too.
